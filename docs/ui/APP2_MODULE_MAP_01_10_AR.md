# خريطة التبويبات والوحدات — App2 — مراحل 01-10

## الوضع الحالي

تم تنفيذ أساس المراحل 01-10 على مستوى البناء والحوكمة وقواعد الأعمال، لكن ليست كل العناصر تحولت إلى تبويبات مستقلة في الواجهة بعد. التطبيق الحالي يستخدم تبويبات رئيسية قليلة، وداخل تبويب المزيد توجد وحدات فرعية.

## التبويبات الرئيسية الحالية

1. الرئيسية
2. أوامر العمل
3. الإشراف والصيانة
4. الأصول
5. المزيد

## الوحدات الموجودة داخل المزيد

### إدارة العمل
- البلاغات
- الصيانة الدورية
- قوالب العمل
- العدادات
- تحليل الأعطال
- CAPA

### الأصول والمواقع
- المواقع الفنية
- الأصول موجودة كتبويب رئيسي
- المستندات والخصائص وBOM وحركة الأصل موجودة داخل بطاقة الأصل

### المخزون والموردون
- المخزون
- المشتريات
- الموردون

### الحوكمة والإدارة
- سجل التدقيق
- سلة المحذوفات
- الإدارة والمستخدمون
- النسخ الاحتياطي والاستعادة داخل شاشة الإدارة
- التقارير

## ما أضيف في مراحل 01-10 وليس تبويبًا مستقلًا بعد

- قواعد الجلسة والصلاحيات
- سياسة كلمة المرور وإنشاء أول مدير
- قواعد حوكمة المخزون
- آلة انتقالات أمر العمل
- فحص النسخة الاحتياطية قبل الاستعادة
- اختبارات الحوكمة

هذه عناصر داخلية تظهر أثرها في السلوك والبناء وليست شاشات منفصلة بالضرورة.

## الترتيب المقترح لتحسين الواجهة في الدفعة التالية

### 1. تبويب المزيد يصبح مجموعات واضحة
بدل شبكة واحدة طويلة، يتم تقسيمه إلى مجموعات:

- العمل والصيانة
- الأصول والمواقع
- المخزون والمشتريات
- الحوكمة والإدارة
- التقارير والنسخ الاحتياطي

### 2. إضافة شاشة مركزية باسم مركز الحوكمة
تعرض:

- حالة الصلاحيات
- حالة النسخ الاحتياطي
- قواعد المخزون
- قواعد إغلاق أمر العمل
- سجل التدقيق
- سلة المحذوفات
- اختبارات البناء الأخيرة إن أضيف ربط لاحقًا

### 3. ترتيب الأصول
داخل بطاقة الأصل يتم ترتيب الأقسام بهذا الشكل:

1. الهوية والموقع
2. الحالة والأهمية
3. المسؤوليات والضمان
4. العدادات والقراءات
5. أوامر العمل والتاريخ
6. المستندات
7. الخصائص الفنية
8. BOM وقطع الغيار
9. الحركة والتركيب
