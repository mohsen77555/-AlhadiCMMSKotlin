# Alhadi CMMS — Kotlin Android App

تطبيق Android Native لإدارة الصيانة والأصول (CMMS) مبني بـ **Kotlin + Jetpack Compose (Material 3) + Room Database**، بواجهة عربية RTL واحترافية على غرار التطبيقات العالمية.

> ملاحظة: مصدر المشروع أصبح الآن في جذر المستودع مباشرةً (بدون ملف ZIP) ليكون قابلاً للبناء فوراً عبر Android Studio وGitHub Actions.

## أبرز المزايا

### واجهة عالمية المستوى (UI/UX)
- نظام تصميم **Material 3** كامل مع ألوان مخصّصة للعلامة (Teal / Indigo).
- **وضع داكن وفاتح** تلقائي + دعم **الألوان الديناميكية** (Material You) على Android 12+.
- شاشة بداية (**Splash Screen**) رسمية وأيقونة تطبيق تكيّفية (Adaptive Icon).
- لوحة تحكم بترويسة متدرّجة الألوان وبطاقات مؤشرات (KPI) بأيقونات ملوّنة.
- **شارات حالة وأولوية ملوّنة دلالياً** (أخضر/أصفر/أحمر/أزرق) عبر كل الشاشات.
- أيقونات احترافية في شريط التنقّل السفلي، وانتقالات حركية ناعمة بين الشاشات.
- نموذج إنشاء أمر عمل عبر **Bottom Sheet** مع اختيار الأصل والأولوية وتاريخ الاستحقاق.

### الحوكمة (Governance)
- **شاشة تسجيل دخول** فعلية مع مصادقة على جدول المستخدمين.
- **صلاحيات حسب الدور (RBAC)**:
  - *Admin*: كل شيء + صفحة الإدارة + سجل التدقيق + إعادة التهيئة.
  - *Supervisor*: إنشاء أوامر عمل + استلام مخزون + إدارة الصيانة.
  - *Technician*: تحديث حالة الأوامر + صرف المخزون + تنفيذ الصيانة الدورية.
- **سجل تدقيق (Audit Log)** يسجّل "مَن فعل ماذا ومتى" لكل إجراء مهم (دخول، أوامر عمل، مخزون، صيانة، مستخدمون).
- إسناد كل عملية للمستخدم المسجّل دخوله فعلياً (وليس مستخدماً افتراضياً).

### الوظائف الأساسية (CRUD كامل في كل قسم)
- **لوحة تحكم** Dashboard مع مؤشرات حيّة وتنبيهات ذكية (أصول حرجة، CAPA، نقص مخزون، صيانة مستحقة).
- **الأصول**: إضافة/تعديل/حذف + بحث وتجميع + **بطاقة تفاصيل** (أوامر العمل والصيانة المرتبطة) + **دورة حياة وحالات** (Running/Stopped/Under Maintenance/Retired...) مع منع الأوامر على الأصل المتقاعد.
- **أوامر العمل**: إنشاء/تعديل/حذف + فلترة وبحث + بدء/إغلاق.
- **الصيانة الدورية**: إضافة/تعديل/حذف + إعادة جدولة تلقائية عند التنفيذ.
- **المخزون وقطع الغيار**: إضافة/تعديل/حذف + حركات صرف/استلام.
- **المستخدمون**: إضافة/تعديل/تفعيل/تعطيل/حذف.

### وحدات حوكمة الأصول المتقدمة (EAM)
- **العدّادات والقراءات**: نقاط قياس لكل أصل (عداد تراكمي/قراءة) + تسجيل قراءات + منع نقصان العداد + تنبيه تجاوز الحد.
- **المواقع الفنية الهرمية**: شجرة مواقع (مصنع → خط → منطقة) + ربط الأصول بها + عرض أصول كل موقع.
- **هرمية الأصول**: أصل رئيسي/فرعي مع التنقّل بين الأصول الأبناء.
- **حركات الأصل (تركيب/نقل/فك/تقاعد)**: تسجيل حركات دورة الحياة مع تحديث الموقع والحالة تلقائياً + **سجل تاريخي (Timeline)** لكل أصل.
- **هوية الأصل والمعلومات المالية**: الرقم التسلسلي + وسم الأصل (Asset Tag) + المورّد + أمر الشراء + تكلفة الشراء + تاريخ الاقتناء (والبحث بها).
- **التكاليف**: تكلفة العمالة (ساعات × أجر) + قطع الغيار لكل أمر عمل + **تجميع التكلفة لكل أصل** والأصول الأعلى تكلفة.
- **الضمان (Warranty)**: حقول ضمان + حالة + تنبيه عند العمل على أصل تحت الضمان + تقرير الضمانات المنتهية.
- **CAPA**: إجراءات تصحيحية ووقائية تغذّي مؤشر CAPA في اللوحة.
- **قوائم الفحص (Inspection Checklists)**: بنود فحص لكل مهمة صيانة دورية مع نتائج (سليم/عطل/لا ينطبق) وتصفير تلقائي عند التنفيذ.
- **سير الاعتماد (Approval Workflow)**: أوامر العمل الحرجة أو عالية التكلفة تتطلب اعتماد المشرف قبل البدء/الإغلاق (اعتماد/رفض).
- **تحليل الأعطال MTTR/MTBF**: متوسط زمن الإصلاح والزمن بين الأعطال لكل أصل، من بيانات أوامر العطل.
- **بطاقة الأصل الشاملة (Asset 360)**: تتضمن — الخصائص الفنية المرنة + المستندات + **رمز QR** + الضمان + المعلومات المالية + بطاقة التكاليف + سجل الحركات + الأصول الفرعية + قائمة المكوّنات **BOM** + أوامر العمل والصيانة المرتبطة + دورة الحياة.
- **التقارير والتحليلات**: التوفّر (Availability) والأعطال وMTTR + حالة أوامر العمل (متأخرة/بانتظار الاعتماد) + التكاليف + المخزون والضمان + **مشاركة التقرير** كنص.
- تشغيل **Offline-first** بالكامل عبر Room (قاعدة بيانات v16) مع سجل تدقيق لكل تغيير.

## فتح المشروع في Android Studio

1. افتح Android Studio واختر **Open**.
2. اختر مجلد المستودع (جذر المشروع حيث يوجد `settings.gradle.kts`).
3. انتظر Gradle Sync ثم شغّل التطبيق على Emulator أو هاتف Android.

## بناء APK

- **من Android Studio:** `Build > Build Bundle(s) / APK(s) > Build APK(s)` → الناتج في `app/build/outputs/apk/debug/`.
- **من GitHub Actions:** يعمل تلقائياً عند الدفع إلى `main`/`master` أو فروع `claude/**`، أو يدوياً من تبويب **Actions** عبر workflow باسم `Build Android APK`. الناتج artifact باسم `Alhadi-CMMS-debug-apk`.
