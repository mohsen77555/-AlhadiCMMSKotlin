# Alhadi CMMS — Kotlin Android App

تطبيق Android Native لإدارة الصيانة والأصول (CMMS) مبني بـ **Kotlin + Jetpack Compose (Material 3) + Room Database**، بواجهة عربية RTL واحترافية على غرار التطبيقات العالمية.

> ملاحظة: مصدر المشروع أصبح الآن في جذر المستودع مباشرةً (بدون ملف ZIP) ليكون قابلاً للبناء فوراً عبر Android Studio وGitHub Actions.

## أبرز المزايا

### واجهة عالمية المستوى (UI/UX)
- نظام تصميم **Material 3** كامل مع ألوان مخصّصة للعلامة (Teal / Indigo).
- **وضع داكن وفاتح** تلقائي + دعم **الألوان الديناميكية** (Material You) على Android 12+.
- شاشة بداية (**Splash Screen**) رسمية وأيقونة تطبيق تكيّفية (Adaptive Icon).
- لوحة تحكم بترويسة متدرّجة الألوان وبطاقات مؤشرات (KPI) بأيقونات ملوّنة.
- **شارات حالة وأولوية ملوّنة دلالياً** (أخضر/أصفر/أحمر/أزرق) عبر كل الشاشات.
- أيقونات احترافية في شريط التنقّل السفلي، وانتقالات حركية ناعمة بين الشاشات.
- نموذج إنشاء أمر عمل عبر **Bottom Sheet** مع اختيار الأصل والأولوية وتاريخ الاستحقاق.

### الحوكمة (Governance)
- **شاشة تسجيل دخول** فعلية مع مصادقة على جدول المستخدمين.
- **صلاحيات حسب الدور (RBAC)**:
  - *Admin*: كل شيء + صفحة الإدارة + سجل التدقيق + إعادة التهيئة.
  - *Supervisor*: إنشاء أوامر عمل + استلام مخزون + إدارة الصيانة.
  - *Technician*: تحديث حالة الأوامر + صرف المخزون + تنفيذ الصيانة الدورية.
- **سجل تدقيق (Audit Log)** يسجّل "مَن فعل ماذا ومتى" لكل إجراء مهم (دخول، أوامر عمل، مخزون، صيانة، مستخدمون).
- إسناد كل عملية للمستخدم المسجّل دخوله فعلياً (وليس مستخدماً افتراضياً).

### الوظائف الأساسية
- لوحة تحكم Dashboard مع تنبيهات ذكية (أصول متوقفة، نقص مخزون، صيانة مستحقة).
- إدارة الأصول مع بحث وتجميع حسب المجموعة.
- أوامر العمل مع فلترة حسب الحالة وتحديث الحالة.
- الصيانة الدورية مع إعادة الجدولة التلقائية.
- المخزون وقطع الغيار مع حركات صرف/استلام.
- تقارير مختصرة + سجل التدقيق.
- تشغيل **Offline-first** بالكامل عبر Room، مع بيانات تجريبية جاهزة.

## فتح المشروع في Android Studio

1. افتح Android Studio واختر **Open**.
2. اختر مجلد المستودع (جذر المشروع حيث يوجد `settings.gradle.kts`).
3. انتظر Gradle Sync ثم شغّل التطبيق على Emulator أو هاتف Android.

## بناء APK

- **من Android Studio:** `Build > Build Bundle(s) / APK(s) > Build APK(s)` → الناتج في `app/build/outputs/apk/debug/`.
- **من GitHub Actions:** يعمل تلقائياً عند الدفع إلى `main`/`master` أو فروع `claude/**`، أو يدوياً من تبويب **Actions** عبر workflow باسم `Build Android APK`. الناتج artifact باسم `Alhadi-CMMS-debug-apk`.
