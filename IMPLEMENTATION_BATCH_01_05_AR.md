# دفعة التنفيذ 01–05

تاريخ التنفيذ: 2026-06-19

## النطاق المنفذ

1. **خط أساس البناء وCI**
   - Gradle Wrapper رسمي لإصدار 8.13 مع SHA-256 مثبت.
   - GitHub Actions لتشغيل فحوص التحقق والاختبارات وlint وبناء Debug وAndroid Test APK.

2. **الاختبارات ومخططات Room**
   - مخططات Room للإصدارات 24 و25 و26.
   - سياسة ترحيل تمنع الحذف المدمر.
   - اختبارات وحدة واختبارات Room/Android للمسارات الحرجة.

3. **أساس Domain والتحقق**
   - أكواد ثابتة للحالات والأدوار بدل الاعتماد على النصوص الحرة في منطق الأعمال.
   - نتائج وأخطاء Domain، مزود وقت، مزود معرفات، وValidators مركزية.
   - `ActionContext` لهوية منفذ العملية.

4. **الإعداد الأول والمصادقة**
   - إزالة إنشاء الحسابات والبيانات التجريبية تلقائيًا.
   - إنشاء أول مدير مرة واحدة عند التشغيل الأول.
   - PBKDF2 مع salt، سياسة كلمة مرور، قفل مؤقت، وتغيير إجباري للاعتماد القديم.
   - ترحيل 24→25 يحفظ المستخدمين القدامى.

5. **الصلاحيات والجلسة والتدقيق**
   - مصفوفة صلاحيات مركزية وفحص الصلاحية داخل Repository.
   - جلسة ذاكرة وإعادة مصادقة للعمليات الحساسة.
   - حماية المستخدم الحالي وآخر مدير نشط.
   - سجل تدقيق غني بهوية مستقرة وbefore/after/reason داخل المعاملة.
   - ترحيل 25→26 يحفظ سجلات التدقيق القديمة.

## التحقق المنفذ

- نجح `scripts/verify_stages_01_05.py`: عدد الفحوص 72.
- نجح `scripts/verify_room_migrations_24_26.py`: تنفيذ 9 أوامر SQL فعلية ومطابقة مخططات Room مع حفظ البيانات القديمة.
- نجح تشغيل Gradle Wrapper 8.13 بعد وضع التوزيع المثبت مسبقًا في cache مؤقت.

## القيد المتبقي قبل الدمج

لم تُسجل اختبارات Gradle وlint وبناء APK كناجحة محليًا؛ بيئة التنفيذ لا تستطيع تنزيل التوزيع من `services.gradle.org` ولا تملك حلًا كاملًا لتبعيات Android في وضع عدم الاتصال. يجب أن تمر مهام GitHub Actions قبل اعتماد الدمج أو الإصدار.
